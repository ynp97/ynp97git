## 2. 現在の開発方針

いまは、アプリそのものを増やすより、各アプリを「どこからでも再開できる状態」に整える時期。

共通方針:

1. まずこのノートに現在地を書く。
2. 実体コードの場所を書く。
3. 起動方法を書く。
4. 次の一手を1つに絞る。
5. Vaultにあるものと、Vault外にあるものを混ぜて扱わない。

POP関連については、送金前クリティカルを進める時期。

優先順位:

1. 出席簿: 保存と復旧を安定させる
2. POP関連: 送金前ゲートと「俺は世界を変える！」の役割を整理する
3. シランガナ: 薄い文字OCRの失敗原因を見える化する
4. Graveyard: 自分用アーカイブとして取り込みと整理を安定させる
5. 元の作業場所と `アプリ本体/` の役割を整理する

開発では、まず「動くものを増やす」より、「何を実行させたいか」を絞る。

## 3. アプリ同士の関係

```text
POP第一期ロードマップ
  ├─ 送金前ゲート
  │    └─ POP送金前ゲート作成補助アプリ
  │         ├─ 家庭ゲート v0.1
  │         └─ 資金性質ゲート v0.1
  │
  └─ 日々の実行
       └─ 「俺は世界を変える！」個人用実行アプリ
            ├─ 今日の一歩
            ├─ 三段階メーター
            ├─ 活動基盤
            └─ 完了履歴

独立アプリ
  ├─ シランガナ
  │    └─ iPhoneカメラOCR
  ├─ Graveyard of the Black Nebula
  │    └─ 音源アーカイブ / 編集机
  └─ 出席簿アプリ
       └─ 学習施設の出席・月次レポート
```

送金前ゲート補助アプリは、本体アプリに入れる前の「実地テスト」。  
本体アプリは、POP第一期全体を日々のタスクへ落とすための土台。

## 5. 他AIへの注意

- まず [[🚀 アプリ開発をどこからでも再開する]] とこのノートを読む。
- いまはアプリ開発そのものを広げすぎない。
- Vault内の記録と、Vault外の実体コードを区別する。
- 実体コードがVault外にある場合、存在確認なしに「編集済み」と言わない。
- POPの送金前判断を進めるために必要な部分だけ触る。
