# 「俺は世界を変える！」アプリ確認シート

## 目的

コードを読まずに、画面と操作で確認するためのシート。

見るべきことは、機能の細部ではなく、次の3点に絞る。

1. POP三段階が見えるか
2. 今日やることが見えるか
3. 完了したときに、何が進んだか分かるか

## 確認方法

`俺は世界を変える！.html` をブラウザで開いて、以下を順番に見る。

## 1. 最初の画面で見ること

- [ ] 「カフェの設立」「孤児院の設立」「いい人システムの確立」の3つが見える
- [ ] 三段階とは別に「活動基盤」が見える
- [ ] 活動基盤が、学校・教会・資金・時間・体力を支えるものだと分かる
- [ ] 学校事業だけのアプリに見えない

## 2. 今日の一歩で見ること

- [ ] 今日やるタスクが表示される
- [ ] タスクの内容が、今のPOP第一期に関係している
- [ ] タスクが大きすぎず、今日か数日で手を付けられそうに見える
- [ ] 「これは今やることではない」と感じるタスクが混ざっていない

## 3. タスクを完了して見ること

テストとして、1つだけ完了してみる。

- [ ] カフェ関連タスクを完了すると、カフェの進捗が動く
- [ ] 活動基盤タスクを完了しても、三段階の数字が水増しされない
- [ ] 完了後のメッセージで、何が進んだか分かる
- [ ] 完了履歴が残る

## 4. POP第一期として見ること

次の流れが、アプリ内のタスクとして見えればよい。

- [ ] 送金と共同経営確認
- [ ] 建設台帳
- [ ] 2026年8月の建設参加・撮影
- [ ] 通常営業の記録
- [ ] 2027年3月の一週間営業
- [ ] 2027年4月の共有・検証・第二期判断

全部が細かく実装されていなくてもよい。今の段階では、「この流れに沿ってタスクを増やせそうか」を確認する。

## 5. 違和感を拾う欄

見ていて違和感があれば、コードではなく言葉で書く。

- 分かりにくい言葉:
- 今やる感じがしないタスク:
- 逆に入れたいタスク:
- 見た瞬間に重く感じるところ:
- 面白くなりそうなところ:

## 確認の結論

最後に、次のどれかを選ぶ。

- [ ] この方向で進める
- [ ] 三段階はよいが、タスクを入れ替える
- [ ] 活動基盤の見せ方を軽くする
- [ ] 今日の一歩をもっと前面に出す
- [ ] まだアプリより、紙の地図を先に整える
