# 「俺は世界を変える！」個人用アプリ仕様

## 目的

大きすぎる目標と、今日できる小さな行動をつなぐ。

タスクを完了したとき、単にチェックが消えるのではなく、**三段階の目標を何パーセント動かしたか**を返し、前進の実感を作る。

## 全体設計 v0.2（大枠）

このアプリは、学校事業の募集管理アプリではなく、POPを進めるための個人用実行アプリとして扱う。学校・教会・資金・時間・体力は、主目標を支える活動基盤として別枠に置く。

確認はコードではなく、[[俺は世界を変える！アプリ確認シート]] を使って、画面と操作で行う。

### 1. 今日の一歩

毎日開いたとき、最初に見る画面。

- 今日やる主タスクを1つ表示する
- 余力があれば触れる短いタスクを1〜2個表示する
- 完了したら、どの段階・どの基盤をどれだけ動かしたかを返す
- 完了履歴を残し、後で素材や記録として使えるようにする

### 2. 三段階メーター

主目標は次の三段階に分ける。

1. カフェの設立
2. 孤児院の設立
3. いい人システムの確立

それぞれを同時に少しずつ進められるが、画面上では別メーターとして表示する。これにより、カフェだけが進んでいるのか、孤児院やいい人システムの準備も進んでいるのかを分けて見られる。

### 3. POP第一期ロードマップ

当面の中心は、POP第一期を日々のタスクに分解すること。

1. 送金と共同経営確認
2. 建設台帳
3. 2026年8月の建設参加・撮影
4. デニスによる通常営業の記録
5. 2027年3月の日本側チームによる一週間営業
6. 2027年4月の共有・検証・第二期判断

このロードマップは、カフェの設立メーターの中心になる。同時に、撮影・参加台帳・結果共有は、いい人システムの小さな実証にもつながる。

### 4. 活動基盤

三段階を進めるための土台として、次の5項目を別メーターで扱う。

- 学校
- 教会
- 資金
- 時間
- 体力

活動基盤のタスクは、三段階の進捗を直接増やさない。三段階の数字を水増ししないため、基盤は基盤として表示する。

学校事業については、細かい募集戦略をアプリの中心にしない。アプリでは「30人を維持できる状態」「現在人数」「卒業見込み」「新規見込み」を入力できる程度に留め、詳しい募集経路や営業計画は別紙で扱う。

### 5. タスクの種類

日々のタスクは、次の3種類に分ける。

- **直接タスク**: カフェ、孤児院、いい人システムを直接進める行動
- **基盤タスク**: 学校、教会、資金、時間、体力を整える行動
- **記録タスク**: 写真、動画、文章、メモ、会計、気づきなどを残す行動

記録タスクは、単体では大きく進まないが、後で発信・共有・クラウドファンディング・検証に使えるため、履歴として残す価値を持たせる。

### 6. まず入れるタスク群

最初に作り込むタスクは、POP第一期に絞る。

- 共同経営確認書の未決事項を埋める
- デニスへ確認する英語メッセージを作る
- 150万円の使途記録フォーマットを作る
- 8月に撮るものリストを作る
- 工事進捗を記録する形式を決める
- 3月の一週間営業でやりたいことを仮置きする
- 参加者に返せるものを洗い出す
- 4月に共有する内容の型を作る

学校事業は、ここでは活動基盤の1項目として最低限の数値入力に留める。

### 7. 次の試作で見るところ

次の試作では、機能を増やすより、次の3点を見る。

- 今日の一歩が自然に選べるか
- 完了したときに、三段階や基盤が動いた実感があるか
- POP第一期のロードマップが、日々の行動に分解されて見えるか

## 三段階

1. **カフェの設立**
2. **孤児院の設立**
3. **いい人システムの確立**

三段階は順番を持つが、完全に直列ではない。カフェを進めながら、孤児院の調査や、いい人システムの小さな実証も並行してよい。

## 活動基盤（自由の確保）

学校事業、教会、資金、時間、体力は、三段階と同列の第四目標にはしない。三段階を実行できる自由を作る**活動基盤**として別枠にする。

- 学校事業の安定と委任
- 教会の安定と委任
- 資金の確保と区分
- 集中できる時間の確保
- 体力の維持

活動基盤のタスクは、カフェ・孤児院・いい人システムの達成率を直接増やさない。その代わり「動ける自由が何パーセント整ったか」を別のメーターで示す。

学校事業については、フリースクール南十字舎とKG高等学院八千代台キャンパスを合わせて**生徒30人を維持できる状態**を具体的な数値目標とする。年間収入が約1000万円となり、人をしっかり雇って南雲の時間を作れる状態を目指す。現在人数は12人。来年は卒業で8人減る見込み。学校事業では毎年卒業が出るため、1年で26人増やすのではなく、2年程度で30人ラインへ近づけ、その後も毎年の卒業数を上回る新規入学を確保する循環モデルで見る。

## 構造

```text
三段階の目標
  └─ 節目
       └─ 日々のタスク

活動基盤
  └─ 学校 / 教会 / 資金 / 時間 / 体力
       └─ 日々のタスク
```

各節目には、段階内での重みを設定する。各タスクにも、その節目の中での重みを設定する。

## 進捗の計算

```text
節目の進捗 = 完了タスクの重み ÷ 登録タスクの重み
段階の進捗 = 各節目の進捗 × 節目の重み
全体の進捗 = 三段階の進捗の平均
```

タスクの数をこなすだけでなく、重要な一歩が大きく反映されるようにする。

## タスク完了時の返答

```text
今日、世界を動かした。
「デニスとの共同経営確認書」を完了。
カフェの設立を 2.4%、全体を 0.8% 動かした。
```

## 最小試作の機能

- 三段階それぞれの進捗表示
- 全体進捗表示
- 活動基盤の進捗表示
- 節目ごとの進捗表示
- 今日やるタスクの表示

## 派生試作: POP送金前ゲート作成補助 v0.0

2026年6月17日に、送金前ゲートを揃えるための独立HTML試作 [[POP送金前ゲート作成補助アプリ.html]] を作成。

これは「揃ったタスクを管理するアプリ」ではなく、家庭ゲート v0.1 と資金性質ゲート v0.1 を作るための入力補助。質問を1つずつ出し、回答を下書きMarkdownにまとめる。

対象:

- 家庭ゲート
- 資金性質ゲート

機能:

- 今日のゲート表示
- 5分／前進／整理の3択
- 質問ごとの回答保存
- 未回答表示
- 家庭ゲート v0.1／資金性質ゲート v0.1 の下書き生成
- Markdownコピー／保存

この試作は、後の「俺は世界を変える！」本体に統合する前の実地テストとして扱う。
- タスクの追加、完了、削除
- タスクの段階、節目、重み、期限の設定
- 完了時に動かしたパーセントを表示
- 今日動かした量と行動履歴
- ブラウザ内への自動保存
- JSONによるデータ書き出し
